# UartToI2c

PIC16F15313を使用したシリアル-I2C変換プログラム。  
シリアルポートからのコマンドでI2CマスタとしてI2Cデバイスを制御します。

開発環境: **MPLAB X IDE + XC8 コンパイラ**

---

## ピン仕様

| ピン | 機能        | 説明 |
|------|-------------|------|
| RA0  | EUSART TX   | シリアル送信（ハードウェア EUSART、PPS 経由）。起動時 9600 bps。RST コマンドで変更可能 |
| RA1  | EUSART RX   | シリアル受信（ハードウェア EUSART、PPS 経由）。起動時 9600 bps。RST コマンドで変更可能 |
| RA2  | RESET       | I2Cデバイスリセット信号。通常時 HIGH。マイコン起動時およびリセットコマンド受信時に 0.5 秒間 LOW |
| RA3  | MCLR        | マスタークリアリセット（MCLRE=ON） |
| RA4  | SCL         | I2C クロック（MSSP ハードウェア I2C、オープンドレイン）。起動時 100 KHz。リセットコマンドで 30〜400 KHz 可変 |
| RA5  | SDA         | I2C データ（MSSP ハードウェア I2C、オープンドレイン） |

## 回路接続

```
                    PIC16F15313
                  +-------------+
  +3.3V / +5V ---|VDD(1)   (8)VSS|--- GND
          SDA ---|RA5(2)   (7)RA0|--- TX --> (USB-UART 変換モジュール RX)
          SCL ---|RA4(3)   (6)RA1|--- RX <-- (USB-UART 変換モジュール TX)
         MCLR ---|RA3(4)   (5)RA2|--- RESET
                  +-------------+
```

### USB-UART 変換モジュールとの接続

| PIC16F15313 | USB-UART 変換モジュール |
|:-----------:|:---------------------:|
| RA0 (TX) | RX |
| RA1 (RX) | TX |
| VSS (GND) | GND |

## シリアルコマンド仕様

### フォーマット

```
<コマンド>,<パラメタ><CR><LF>
```

コマンド処理完了後に `OK\r\n` を、エラー時は `NG,<code>\r\n` を返します。

| エラーコード | 意味 |
|---|---|
| `NG,CM` | コマンド構文エラー、値の範囲外 |
| `NG,I2` | I2C 送受信エラー |
| `NG,CS` | SND チェックサム不一致 |

---

### コマンド一覧

#### 1. バージョン取得 `VER`

```
VER<CRLF>
```

現在の ROM バージョンを返します。

**レスポンスフォーマット:**
```
<バージョン文字列>\r\n
OK\r\n
```

**例:**
```
VER
UartToI2c Ver1.00
OK
```

---

#### 2. NOP `NOP`

```
NOP<CRLF>
```

何も行わず `OK\r\n` を返します。

---

#### 3. I2C リセット `RST`

```
RST[,<オプション1>[@<オプション2>]]<CRLF>
```

オプションは `@` で区切り、順不同でいずれも省略可能です。

| オプション | 説明 |
|------------|------|
| 数値       | I2C 速度（KHz 単位）。30-400KHz。省略時は現在の速度を引き継ぐ |
| `NR`       | RESET(RA2) 信号の立ち下げ（LOW）および立ち上げ（HIGH）を行わない。500ms の待機は実施する |
| `S1`-`S8`  | シリアル通信速度。S1:9600bps,S2:19200bps,S3:38400bps,S4:57600bps,S5:115200bps,S6:230400bps,S7:460800bps,S8:921600bps |

- 存在しないオプションを指定した場合はリセット動作を行わず `NG,CM\r\n` を返す

**動作:** RESET=LOW（`NR` 指定時はスキップ）→ 500 ms 待機 → RESET=HIGH（`NR` 指定時はスキップ）→ 200 ms 待機 → 指定速度で I2C 再初期化 → シリアル速度切替（`S1`-`S8` 指定時のみ）→ `OK\r\n`（新速度で送信）

> **注:** オプション解析エラー時の `NG,CM\r\n` は変更前の速度で送信されます。

**例:**
```
RST,30@NR@S5
OK
```
（I2C通信速度 30 KHz、RESET 信号立ち下げ無し、シリアル通信速度 115200 bps）

```
RST,400
OK
```
（I2C 通信速度を 400 KHz に変更）

```
RST,S8
OK
```
（I2C 通信速度は直前の設定を引き継ぎ、シリアル通信速度を 921600 bps に変更）

上記 2 コマンド実行後: I2C=400 kbps、シリアル=921600 bps

#### 4. I2C 送信 `SND`

```
SND,<スレーブアドレス>,<データ>[,<NS|チェックサム>]<CRLF>
```

- `<スレーブアドレス>`: 00〜7F（16進数 2 桁）
- `<データ>`: 16進数 2 桁の連続。最大 66 バイト（132 文字）
- `<NS|チェックサム>`: 以下のいずれかを省略可能で指定
  - `NS` : ストップコンディションを送信しない（継続フラグ）。次の SND コマンドではスタートコンディションとスレーブアドレスを送信せずにデータのみ送信する。チェックサムフィールドとは排他
  - チェックサム : **トランザクション全体**（NS 継続中の全チャンクを含む）のデータバイト列に対する CRC-8 チェックサム（1 バイト = 16進数 2 桁）。最終の SND コマンド（NS なし）にのみ指定可能

**チェックサム計算:** CRC-8 をデータバイト列全体に対して計算した 1 バイト値を使用します。パラメータは以下のとおりです。

- 多項式: x^8 + x^2 + x + 1 (`0x07`)
- 初期値: `0x00`
- ビット順: MSB-first
- 入力反転（RefIn）: なし
- 出力反転（RefOut）: なし
- 最終 XOR（XorOut）: `0x00`

チェックサムが一致しない場合は `NG,CS\r\n` を返し、I2C 送信は行いません（バスがオープンの場合は STOP を送信してからエラーを返します）。チェックサムを省略した場合は検証を行いません。

**NS フラグによる 130 バイト送信例（チェックサムあり）:**

```
SND,7F,<66バイト分の16進132文字>,NS
OK
SND,7F,<64バイト分の16進128文字>,<全130バイトのCRC-8>
OK
```

1 回目の SND は I2C スタート＋アドレス＋66 バイトを送信し、CRC-8 の途中計算値を保持してストップを送信せずに OK を返します。  
2 回目の SND はスタート・アドレスなしで残り 64 バイトを送信し、全 130 バイトに対する累積 CRC-8 を検証してからストップを送信します。

**動作:** 累積 CRC-8 計算（初回は 0 から、継続時は前回の計算結果から）→ チェックサム検証（指定時のみ）→ スタート（g_i2c_open=false 時のみ）→ アドレス（Write）（g_i2c_open=false 時のみ）→ データ順次送信 → NS 指定時: ストップなし・CRC 状態保存・OK / 非指定時: CRC 状態クリア・ストップ → 全 ACK なら `OK\r\n`、NACK は `NG,I2\r\n`、チェックサム不一致は `NG,CS\r\n`

> **メモリ上の制約について**  
> PIC16F15313 の RAM は 256 バイトです。`SND` 実装はコマンドバッファ上の 16 進文字列を直接ストリーミング解析して I2C 送信するため、別途データ配列を持ちません。  
> バッファサイズ（`CMD_BUF_SIZE = 144`）からコマンドオーバーヘッド（11 バイト）を引いた上限は **66 バイト**（`I2C_DATA_MAX`）です。NS フラグを使うと複数の SND コマンドに分割して 1 つの I2C トランザクションで送信でき、コマンドごとの上限（66 バイト）はそのまま維持されます。合計バイト数はスレーブデバイスのバッファ容量と I2C プロトコルの制約に依存します。

**例:**
```
SND,50,00002F2A,XX
OK
```
（`XX` はデータ `00 00 2F 2A` の CRC-8 値）

#### 5. I2C 受信 `RCV`

```
RCV,<スレーブアドレス>,<受信バイト数><CRLF>
```

- `<スレーブアドレス>`: 00〜7F（16進数 2 桁）
- `<受信バイト数>`: 1〜8192

**動作:** スタート → アドレス（Read）→ 受信しながら 16 進 2 桁を随時シリアル出力 → ストップ → カンマ `,` → 受信バイト列全体の CRC-8 チェックサムを 16 進 2 桁で出力 → `\r\n` → `OK\r\n`  
各バイト受信時に 10ms 以内にデータが届かない場合はタイムアウトと判定し、`NG,I2\r\n` を返します。

**レスポンスフォーマット:**
```
<データ>,<チェックサム>\r\n
OK\r\n
```

**タイムアウト時レスポンス:**
```
NG,I2\r\n
```

**例:**
```
RCV,01,10
0A1B2C3D4E5F60718293,XX
OK
```
（`XX` は受信データ全体の CRC-8 値を 16 進 2 桁で表したもの）

## 起動シーケンス

1. ポート初期化（RESET=LOW）
2. I2C（MSSP ハードウェア I2C）初期化（100 KHz）
3. UART 初期化（9600 bps）
4. RESET を 0.5 秒 LOW
5. RESET=HIGH
6. 200 ms 待機
7. I2C バスリカバリー（スタック状態のスレーブをフラッシュ）
8. `OK\r\n` 送信 → シリアルコマンド待ちループへ

## 技術メモ

- 内部オシレータ 32 MHz 使用
- MCLRE=ON（RA3 = MCLR）、LVP=ON（通常の ICSP プログラミング使用可能）
- I2C はハードウェア MSSP（RA4=SCL1、RA5=SDA1）。PPS 設定: SSP1CLKPPS=0x04（RA4 → SCL 入力）、SSP1DATPPS=0x05（RA5 → SDA 入力）、RA4PPS=0x15（SCL1 出力）、RA5PPS=0x16（SDA1 出力）。TRIS=1（入力）＋ ODCONA=1（オープンドレイン）。外部プルアップ必要
  - I2C 速度設定: SSP1ADD = 32000 / (4 × speed_kbps) − 1（SSP1STAT: 100kbps 以下 SMP=1、100kbps 超 SMP=0）
  - タイムアウト: `I2C_Wait()` / `I2C_ReadWait()` は 10ms タイムアウト（`g_i2c_1ms_loop` = FOSC/4/1000 = 8000 ループ/ms）
  - バスリカバリー（`I2C_Recovery()`）: SSPEN=0 → SSP1CON2=0 → TRISA[4,5]=出力 → SCL 最大 9 クロック → STOP 擬似生成 → TRISA[4,5]=入力 → WCOL/SSPOV クリア（SSP1CON1 &= ~0xC0）→ SSPEN=1 → BF 残留時 SSP1BUF 読み捨て
  - `PPS1WAY=OFF`: バスリカバリー中に PPS を一時的に再設定できるよう複数回書き込みを許可
- UART はハードウェア EUSART（RA0=TX、RA1=RX）。PPS 設定: RA0PPS=0x0F（TX1 出力）、RX1DTPPS=0x01（RA1 入力）。起動時 9600 bps。RST コマンドの S1〜S8 オプションで変更可能
  - SPBRG 値（FOSC=32MHz, BRG16=1, BRGH=1）:
    - desired: 9600bps, spbrg: 832, actual: 9604bps, error: 0.04%
    - desired: 19200bps, spbrg:416, actual: 19185bps, error: 0.08%
    - desired: 38400bps, spbrg:207, actual: 38462bps, error: 0.16%
    - desired: 57600bps, spbrg:138, actual: 57554bps, error: 0.08%
    - desired: 115200bps, spbrg:68, actual: 115942bps, error: 0.64%
    - desired: 230400bps, spbrg:34, actual: 228571bps, error: 0.79%
    - desired: 460800bps, spbrg:16, actual: 470588bps, error: 2.08%
    - desired: 921600bps, spbrg:8, actual: 888888bps, error: 3.68%
